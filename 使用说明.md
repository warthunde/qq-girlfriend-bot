## 🚀 快速开始（三步搞定）

### 前置条件

- ✅ 已安装 Docker 和 Docker Compose
- ✅ 有一个 QQ 小号（用于登录机器人）
- ✅ 有 DeepSeek API Key（[免费注册获取](https://platform.deepseek.com)，新用户送 500 万 tokens）

### 第一步：解压部署包

将 `qq_bot_deploy.zip` 解压到任意目录。

### 第二步：运行部署脚本

**Windows（PowerShell）：**

```powershell
cd qq_bot_deploy
.\deploy_qq_bot.ps1
```

> 如果提示执行策略错误，先运行：
> ```powershell
> Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
> ```

**Mac / Linux（终端）：**

```bash
cd qq_bot_deploy
bash deploy_qq_bot.sh
```

### 第三步：扫码登录

脚本运行完成后，会提示你：

1. 打开浏览器访问 **http://localhost:6099**（NapCat Web UI）
2. 在页面中找到二维码
3. 使用**手机 QQ** 扫描二维码登录

> ⚠️ **重要**：请使用 QQ **小号**扫码，不要使用大号！

扫码完成后，你的赛博女友就上线了！给她发一条消息试试吧 💬

## 📱 部署后操作

### 验证机器人是否在线

1. 打开 http://localhost:6185（AstrBot Web 管理后台）
2. 在仪表盘查看连接状态
3. 绿色 = 正常在线，红色 = 连接异常

### 测试机器人

用你**自己的 QQ 号**（大号）给机器人 QQ（小号）发一条私聊消息，看看她怎么回复。

也可以把机器人拉进一个群，在群里 @ 她或直接聊天。

### 查看日志

```bash
# 查看 AstrBot 日志（AI 对话引擎）
docker logs -f qq_bot_astrbot

# 查看 NapCat 日志（QQ 协议层）
docker logs -f qq_bot_napcat

# 按 Ctrl+C 退出日志查看
```

## 🌐 Web 管理后台使用

AstrBot Web 管理后台地址：**http://localhost:6185**

### 主要功能

| 功能 | 说明 |
|------|------|
| **仪表盘** | 查看机器人运行状态、连接状态、消息统计 |
| **AI 模型设置** | 切换模型（DeepSeek / OpenAI）、修改 API Key |
| **人格设定** | 修改 System Prompt，自定义角色性格 |
| **行为设置** | 调整回复延迟、打字效果、群聊规则 |
| **插件管理** | 安装/卸载插件，扩展机器人功能 |
| **日志查看** | 实时查看对话日志和错误日志 |
| **数据统计** | 查看消息量、Token 消耗等 |

### 首次配置建议

1. 进入「AI 模型设置」确认 API Key 和模型名称正确
2. 进入「行为设置」调整回复延迟（推荐 3~8 秒）和打字效果
3. 进入「人格设定」预览或修改 System Prompt

## 🎭 修改人格设定

### 方法一：通过 Web 后台（推荐）

1. 打开 http://localhost:6185
2. 进入「人格设定」页面
3. 直接在文本框里修改 System Prompt
4. 点击保存

### 方法二：通过配置文件

编辑 `~/astrbot/config/astrbot_config.yaml`，找到 `personality.system_prompt` 字段：

```yaml
personality:
  system_prompt: |
    你的名字是「XXX」...
    在这里写入新的人格设定...
```

修改后重启容器：

```bash
cd ~/astrbot
```

## 🔧 常见问题排查

### 问题 1：镜像拉取失败

**现象**：`Error response from daemon: Get https://registry-1.docker.io/v2/...`

**原因**：Docker Hub 在国内访问不稳定。

**解决方案**（任选一种）：

**方案 A：使用脚本自带的多代理自动切换**
脚本已内置 5 个国内镜像代理，会自动尝试，无需手动干预。

**方案 B：手动配置 Docker 镜像加速器**

编辑 Docker 配置文件：

- **Windows/Mac**：Docker Desktop → Settings → Docker Engine → 添加 registry-mirrors
- **Linux**：编辑 `/etc/docker/daemon.json`

```json
{
  "registry-mirrors": [
    "https://dockerproxy.com",
    "https://docker.m.daocloud.io",
    "https://dockerhub.azkubernetes.cn"
  ]
}
```

然后重启 Docker 服务。

**方案 C：使用 VPN / 代理**
开启全局代理后再运行脚本。

---

### 问题 2：QQ 扫码后登录失败或掉线

**现象**：扫码成功但显示离线，或登录后很快掉线。

**可能原因与解决**：

1. **滑块验证**：登录时可能需要滑块验证。在 NapCat Web UI (localhost:6099) 中完成验证。
2. **设备锁**：QQ 可能检测到异地登录。在手机 QQ 中确认登录即可。
3. **账号风控**：新设备登录可能被风控。建议：
   - 使用常用网络（不要用代理/VPN）
   - 先在手机 QQ 上把账号挂一段时间
   - 避免频繁重新登录
4. **查看详细日志**：`docker logs qq_bot_napcat`

---

### 问题 3：端口被占用

**现象**：`Bind for 0.0.0.0:6185 failed: port is already allocated`

**解决方案**：

```bash
# 查看占用端口的进程
# Windows:
netstat -ano | findstr 6185

# Mac/Linux:
lsof -i :6185
# 或
ss -tlnp | grep 6185

# 停止占用端口的进程，或修改 docker-compose.yml 中的端口号
```

---

### 问题 4：机器人不回复消息

**检查清单**：

1. ✅ NapCat 是否在线？（访问 http://localhost:6099 确认）
2. ✅ AstrBot 是否连上 NapCat？（查看 AstrBot 日志）
3. ✅ API Key 是否正确？（Web 后台 → AI 模型设置）
4. ✅ API 额度是否用完？（登录 DeepSeek 平台查看余额）
5. ✅ 是否被风控？（尝试给机器人发私聊消息而非群聊）

**查看 AstrBot 日志定位问题**：

```bash
docker logs -f qq_bot_astrbot
```

常见错误提示：
- `401 Unauthorized` → API Key 无效
- `402 Payment Required` → API 余额不足
- `Connection refused` → NapCat 未运行或未连上
- `Context length exceeded` → 对话历史过长，清空上下文

---

### 问题 5：机器人回复太慢

**原因与解决**：

1. **DeepSeek API 服务器负载**：高峰期可能较慢，等待即可
2. **回复延迟设置过长**：Web 后台调整延迟范围（建议 3~8 秒）
3. **网络问题**：检查到 `api.deepseek.com` 的网络延迟

---

### 问题 6：容器频繁重启

**查看重启原因**：

```bash
docker ps -a --filter name=qq_bot
# 查看 STATUS 列，如果频繁 Restarting

docker logs --tail 50 qq_bot_astrbot
# 查看最近的错误日志
```

常见原因：
- 配置文件格式错误（YAML 缩进问题）
- API Key 格式无效
- 内存不足（Docker Desktop 分配内存太少）

## 🛠 高级配置

### 自定义 AI 模型

除了 DeepSeek，还支持以下 AI 提供商：

| 提供商 | 模型示例 | 配置方式 |
|--------|----------|----------|
| **DeepSeek** | `deepseek-chat`（推荐） | 默认配置 |
| **DeepSeek** | `deepseek-reasoner` | 推理模型（更慢但更深思熟虑） |
| **OpenAI** | `gpt-4o` | 填 OpenAI API Key |
| **OpenAI** | `gpt-4o-mini` | 更便宜，更快 |
| **其他兼容接口** | 任意 OpenAI 兼容 API | 修改 Base URL |

**切换模型**：

1. 打开 Web 后台 → AI 模型设置
2. 选择提供商和模型
3. 填入对应的 API Key

**使用自定义 API 代理**：

如果你的 DeepSeek API 走代理（如 `api.qq.com/v1` 之类的兼容接口），修改 `.env` 文件：

```
DEEPSEEK_BASE_URL=https://your-proxy-url.com
```

### 接入更多平台

AstrBot 支持多平台接入（通过安装插件）：

- **QQ 群/私聊**（已配置，NapCat）
- **Telegram**（需安装 Telegram 插件）
- **Discord**（需安装 Discord 插件）
- **微信**（需安装微信插件，较复杂）

在 Web 后台 → 插件管理 → 安装对应插件。

### 群聊白名单/黑名单

编辑 `.env` 文件：

```bash
# 只允许这些群使用机器人（留空 = 所有群）
ALLOWED_GROUP_IDS=123456789,987654321

# 禁止这些群使用机器人
BLOCKED_GROUP_IDS=111111111
```

修改后重启：`docker compose restart`

```bash
# 备份整个项目目录
tar -czf astrbot_backup_$(date +%Y%m%d).tar.gz ~/astrbot

# 恢复备份
```

## 📋 日常维护命令

```bash
# ========== 查看状态 ==========

# 查看容器运行状态
docker ps -a --filter name=qq_bot

# 查看资源使用
docker stats qq_bot_astrbot qq_bot_napcat

# ========== 日志查看 ==========

# 实时查看 AstrBot 日志
docker logs -f qq_bot_astrbot

# 查看最近 100 行日志
docker logs --tail 100 qq_bot_astrbot

# 查看 NapCat 日志
docker logs -f qq_bot_napcat

# ========== 重启/停止 ==========

# 重启所有服务
cd ~/astrbot && docker compose restart

# 只重启 AstrBot
cd ~/astrbot && docker compose restart astrbot

# 停止所有服务
cd ~/astrbot && docker compose down

# 启动所有服务
cd ~/astrbot && docker compose up -d

# ========== 更新升级 ==========

# 拉取最新镜像并重启
cd ~/astrbot && docker compose pull && docker compose up -d

# 查看当前使用的镜像版本
docker images | grep -E "astrbot|napcat"

# ========== 故障恢复 ==========

# 完全重置（保留配置文件）
cd ~/astrbot && docker compose down
docker system prune -f
docker compose up -d

# 完全重置（删除所有数据，谨慎操作！）
cd ~/astrbot && docker compose down -v
rm -rf ~/astrbot/napcat/data ~/astrbot/astrbot/data
docker compose up -d

# ========== 进入容器调试 ==========

# 进入 AstrBot 容器
docker exec -it qq_bot_astrbot sh

# 进入 NapCat 容器
docker exec -it qq_bot_napcat sh
```

## 🏗 架构说明

```
┌─────────────────────────────────────────────────────┐
│                    你的 QQ 小号                       │
│         (手机 QQ 扫码登录，作为机器人账号)            │
└──────────────────┬──────────────────────────────────┘
                   │ QQ 协议
                   ▼
┌─────────────────────────────────────────────────────┐
│              NapCat (Docker 容器)                     │
│  - QQ 协议层，负责收发 QQ 消息                        │
│  - WebSocket 服务器 (端口 3001)                       │
│  - Web UI 管理界面 (端口 6099，扫码用)                │
└──────────────────┬──────────────────────────────────┘
                   │ WebSocket (ws://napcat:3001)
                   ▼
┌─────────────────────────────────────────────────────┐
│             AstrBot (Docker 容器)                      │
│  - AI 对话引擎                                        │
│  - 连接 DeepSeek API 进行智能回复                     │
│  - Web 管理后台 (端口 6185)                           │
│  - 人格管理 / 插件系统 / 日志记录                     │
└──────────────────┬──────────────────────────────────┘
                   │ HTTPS
                   ▼
┌─────────────────────────────────────────────────────┐
│             DeepSeek API (云端)                       │
│  - deepseek-chat 模型                                 │
│  - 根据 System Prompt 生成角色回复                    │
└─────────────────────────────────────────────────────┘
```

### 网络说明

- NapCat 和 AstrBot 通过 Docker 内部网络通信（`napcat:3001`）
- 对外只暴露 Web 管理端口（6099、6185）
- 用户通过 Web UI 扫码和管理，不需要直接操作 QQ 客户端
